DNS服务器

2019-12-30

CVE _2019-12-31

DNS

正向解析：将域名解析成IP地址，A记录 CNAME记录 MX记录

反向解析：将IP地址解析成域名，PTR指针

DNS的查询方式：递归查询 迭代查询

区域类型：hint 根区域

​ master 主要区域

slave    辅助区域

配置DNS服务器:以配置www. zlt. com为例
1.打开：/etc/named.rfc1912.zones，配置区域

zone “zlt.com.” IN {
type master;
file “zlt.com.zone”;
allow-update { none; };
};

2.到/var/named目录下配置区域配置文件

cp named.localhost zlt.com.zone

chown :named zlt.com.zone

3.修改区域配置文件的所属组为named
4.启动服务/重启服务
5.设置DNS地址
6.测试DNS：nslookup

假设有另外一个公司的网站也放在这台服务器上，域名为www.aa.com
1.打开：/etc/named.rfc1912.zones，配置区域

zone “aa.com.” IN {
type master;
file “aa.com.zone”;
allow-update { none; };
};

2.到/var/named目录下配置区域配置文件

cp named.localhost aa.com.zone

chown :named aa.com.zone

$TTL 1D
@ IN SOA aa.com. root.aa.com. (
0 ; serial
1D ; refresh
1H ; retry
1W ; expire
3H ) ; minimum
NS dns.aa.com.
dns A 192.168.75.158
www A 192.168.75.158
ftp A 192.168.75.158
A 127.0.0.1
AAAA ::1

3.启动服务/重启服务

配置辅助DNS服务器
1）安装bind
2）启动bind
service named start
chkconfig named on
3）修改主配置文件，添加辅助区域
vim /etc/named.conf

zone "zlt.com" IN {
       type slave;
       file "slaves/slave.com.zone";
       masters {192.168.0.1;};
    };

zone "0.168.192.in-addr.arpa" IN {
       type slave;
       file "slaves/0.168.192.zone";
       masters {192.168.0.1;};
    };

4)重启服务：service named restart
